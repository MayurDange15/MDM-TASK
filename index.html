Dynamic Residue Network analysis is calculated from the aggregation of metrics computed from network graphs, which are themselves computed over each frame from a conformational sampling experiment, typically MD. Each graph consists of nodes (protein C-beta, or GLY C-alpha atoms) connected by edges that are assigned using a user-defined cut-off distance for each protein residue pair. The default cut-off distance of 6.7 Angstroms comes from previous work identifying the first of a series of coordination shells using the radial distribution function in a collection of proteins. This value can be changed if desired, for experimentation.
For faster uploads we recommend reducing the input trajectory.

Aggregation methods

The DRN can be computed as an average or as a median across all frames. The standard deviations can also be computed.

Available DRN metrics

  • Betweenness Centrality (BC)
    • BC is the fraction of times a node is found within the shortest path connecting each node pair in a network graph, out of all the shortest paths in the same graph.
  • Averaged Shortest Path Lengths (L)
    • L is computed as as the average of the shortest path lengths from every other node to a given node, in a graph.
  • Closeness Centrality (CC)
    • CC is the inverse of L.
  • Eccentricity (ECC)
    • ECC is obtained from the averaged longest path lengths.
  • Degree Centrality (DC)
    • DC is the connectivity of a node, obtained by counting its closest neighbors.
  • Eigencentrality (EC)
    • EC is obtained as the largest eigenvalue's corresponding eigenvector when the adjacency matrix is decomposed. The obtained eigencentrality then is a representation a node's centrality based on the centrality of its neighbors.
  • Katz centrality (KC)
    • KC is a generalization of EC, which via an adjacency damping coefficient alpha and a basal adjacency beta, assigns centrality on the basis of the node’s immediate connectivity.
  • PageRank (PR)
    • PR is an adjusted version of KC, which assigns centrality values by factoring in the degree of their neighbors and the importance of their connections.

DRN options

These options are a variety of network centrality metrics that can be applied to a representation of a single protein structure. Network centrality are different ways of investigating node importance in a network. As they are computed from a single structure, they are not averaged.

Contact cutoff distance

This is the cut-off distance in Angstroms used to infer residue contacts. We recommend the default value of 6.7 Angstroms. While smaller or larger values may work in simpler calculations such as DC, convergence problems may be experienced for very different values for metrics based on shortest path calculations and those using matrix factorization, such as EC. Larger cut-off values will also increase computation time, as more edges need to be traversed. DRN statistic

Three options are provided as summary statistics for residue centralities computed across each frame, namely the average, median and standard deviation.

Perturbations

This represents the number of minute random uniformly distributed forces applied to each CA atom to drive overall motions. Each residue is scanned with this approach. By comparing the similarity of the perturbed conformations to that of a desired conformation, one can determine hotspot residues that drive conformational changes associated with some defined state.

Large protein complexes can be difficult to study with limited hardware resources. Coarse-graining therefore allows one to study the motion of large protein complexes such as capsids, by recapitulating the slow modes from the reduced number of atoms. The coarse graining approach preprocesses the input protein structure to obtain a uniform distribution of atoms, before proceeding to the anisotropic network model construction. The coarse graining level can be tuned depending on the problem at hand to control the number.Additionally, one can select the type of residue atom to consider for them network model. A coarse graining level of one filters the data only on the basis of atom type.

Calculates the normal modes using the Anisotropic Network Model, modeling interatomic interactions as Hookean springs. By diagonalisating the Hessian matrix, which is a matrix of spring constants, it solves for the eigenvectors corresponding to the modes of motion and their associated frequencies (eigenvalues) for a protein complex.

Further details about theory of ANM calculation are provided here.

Protein structure

The protein structure can be a previously coarse-grained protein in the "Coarse Graining" step, or one that has not been goarse-grained.

Cut-off distance

The cut-off distance in Angstroms is used to infer residue contacts. A larger number will retain more atoms for the calculation of the ANM model. Smaller cut-off values may result in too sparse and inaccurate protein representations. A reasonable value will be partly informed by the presence of six leading eigenvalues associated to the resulting normal modes.

The hessian matrix has been decomposed into 3 matrices, namely two for the left (U) and right (VT) singular vectors, together with a diagonal matrix of eigenvalues (W), using the singular value decomposition method. The current tool utilises the rows (eigenvectors) of the VT matrix and the mode number (the row) and displays it in the visualiser, overlaid onto the provided protein structure.

The residue mean square fluctuation is computed from W and VT matrices.

General Features

  • Performs essential dynamics on an MD trajectory, but performs comparative ED if two trajectories are provided.
  • Internally, the two trajectories are aligned to a frame, before decomposing the covariance matrix to obtain a common spatial representation.
  • C- and N-terminus exclusion can increase the performance of principal component axes. Percentages of explained variance are shown in the axes.
  • PCA can be performed on a selection (e.g. residues of a domain and/or chains).
  • The conformation at the energy basin is automatically extracted from the combined ED/KDE plot. The K-means algorithm is also coupled as a separate method of conformation extraction from the PC centroids. This has the ability to sample conformations that may not be at the lowest energy minima within the sampled conformations. General detail of PCA are given here.

Comparative topology / trajectory

If another topology and trajectory pair is provided, the tool will perform comparative essential dynamics. One important requirement is that the data used should be comparable (same number of alpha carbon atoms and number of MD frames)

Selection

Perform ED using the specified selection (MDTraj selection syntax) Only CA atoms will be selected from the selection. It is thus most applicable when specifying residue(s) and/or chain ID(s). Selection is performed after having done a global fitting of the entire structure.

Selection examples:

Select residues 1-10: 'residue 1 to 10 residues'

Select the first chain: 'chainid 0'

Select the residues from 1st and second chains: 'chainid 0 and (residue 10)'

Step size

This parameter controls the frame sampling rate from the MD trajectory. A larger number will result in faster calculations at the expense of accuracy.

Number of clusters

Expected number of clusters for automated conformation extraction using k-means clustering. This has no impact on the essential dynamics - only on the predicted possible equilibrium conformations. This algorithm is used alongside another one which already finds the conformation of maximal probability density.

No. of N- and C-terminus residues to ignore:

By ignoring a certain number of residues at the termini, the performance of decomposition can be improved. Independent analysis of RMSF may assist in setting these parameters. From our observations, this generally affects the last 2-3 residues of the C-termini. The removal is applied the same way to each chain of a complex. Sometimes these changes may be relevant - knowledge about the protein always helps.

Performs essential dynamics from an MD trajectory by decomposing the covariance matrix obtained from the reshaped cartesian coordinates composed only of C-alpha atoms, after aligning each frame to the first frame. By decomposing the matrix of coordinates and plotting the first few principal components (contributing to the largest amount of the total variance) against each other, one can have an idea of the similarities and differences in sampled conformations according to the PCs. The percentage of the explained variation captured by each orthogonal axis is shown in the x and y axes, while the markers from the scatter plots are colored according to the frame number. Pairwise communication propensity (the mean-square fluctuation of the inter-residue distance) is based on the fact that intra-protein signal transduction events are directly related to the distance fluctuations of communicating atoms (Chennubhotla and Bahar, 2007; Penkler and Taştan Bishop, 2019). Low CP values correspond to more efficient (faster) communication compared to larger values. The NxN CP matrix is represented as a heat map.

In this example, a closed conformation HIV protease variant was simulated by MD. An opened-conformation of the protease was then retrieved from the PDB to be used as the target conformation. The PRS algorithm perturbs each residue from the input topology/trajectory via by independently multiplying its covariance matrix against a series of randomly generated force vectors that each contain zeroes, except for the residue C-alpha carbon atoms being targeted. This perturbation is repeated for each residue. The similarity between the perturbed state and the target state is measured and reported as a correlation. In the figure below the correlations default to a range of [0, 1], but the range can also be selected based on the dataset's minimum and maximum, which are also shown. As the opened conformation is characterised by a larger distance between the flap tips, it can be observed that the regions that have a redder tinge are either part of the flap or have a close connection with them. The result becomes more pronounced when the min-max scaling button is pressed.

In this example, the human Enterovirus 71 capsid pentamer obtained from 3VBS has been coarse-grained according to the method described here. This step was applied to reduce the number of atoms before proceeding to the construction and decomposition of the Hessian matrix to obtain the normal modes of motion. This coarse-graining step reduces the protein into a uniform distribution of atoms (C-beta or C-alpha) representative of each residue, thus facilitating computation. Depending on the computational tractability of ANM, this step may not always be needed, but can be employed to speed up calculations for very large complexes by using larger values for the coarse-graining level. In this example, two HIV protease variants bound to the drug darunavir were each simulated by MD. A single PCA run was performed, aligning all the frames to the first frame from one of the variants. Three C-terminus residues were ignored (default) prior to alignment, and resulted in a higher proportion of explained variance for the 1st two principal axes obtained from the covariance matrix obtained from the combined data from both complexes. K-means centroids are shown as red circled fonts, while the conformation from the lowest energy basin is in blue. It should be noted that the k-means algorithm is non-deterministic, even though sampling is repeated a moderately large number of times to mitigate this effect. The contour plot corresponds to the kernel densities of the first two principal components, and is comparable to the free energy surface. All conformations are readily available from the zipped file, for further analysis.
